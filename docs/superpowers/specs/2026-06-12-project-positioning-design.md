# A/B Test Workbench 项目定位设计

## 一. 推荐定位

项目主定位为：**产品数据工具 / 增长分析工具**。

项目不只展示一个统计脚本，而是展示将 A/B 测试分析流程产品化的能力：把 Excel 导入、字段识别、显著性检验、质量 Gate、图表生成和问题上报整合为一个本地 Web 工作台，降低业务团队使用实验分析工具的门槛。

## 二. 项目标题

推荐用于简历和 GitHub 的标题：

**A/B Test Workbench：产品增长实验分析工作台**

## 三. 一句话介绍

一个面向产品增长场景的本地 A/B 测试分析工作台，将 Excel 数据导入、字段识别、显著性检验、质量 Gate、图表生成和问题上报整合为可复用的单页工具。

## 四. 核心价值

原始流程依赖脚本、Excel 文件和人工解释，使用门槛较高，也容易遗漏数据质量风险。本项目把该流程收敛到一个本地网页工具中，让用户可以按固定路径完成：

```text
上传实验数据 -> 识别字段 -> 选择分组和指标 -> 执行显著性分析 -> 查看结论和图表 -> 生成问题报告
```

该定位同时体现三类能力：

- 产品理解：把复杂分析流程转化为可操作的工作台。
- 数据分析：输出 p 值、置信区间、提升幅度、显著性和质量 Gate。
- 工程落地：使用 React + Vite + FastAPI 构建前后端联动工具，并通过本地文件系统管理上传、结果和图表。

## 五. 简历亮点

可在简历项目经历中突出：

- 设计并实现本地 A/B 测试分析工作台，支持 Excel 上传、sheet 识别、表头识别和字段预览。
- 封装实验分析流程，支持 1 个分组字段和 1 到 5 个分析指标，输出 A/B 值、差异、p 值、置信区间、样本量和 PM 可读结论。
- 引入 `PASS` / `PASS_WITH_WARNING` / `FAIL` 质量 Gate，避免把数据质量风险误包装为确定结论。
- 支持指标图、汇总图和完整图表合集生成，便于将实验结果沉淀为可分享材料。
- 基于 React + Vite + FastAPI 构建本地 Web 工具，适合处理不便上传到外部系统的业务数据。

- 如何本地启动。
- 如何用示例 Excel 快速体验。

推荐 README 结构：

1. 项目简介
2. 核心功能
3. 产品流程截图或流程图
4. 技术栈
5. 本地启动

7. 项目结构

## 七. 范围边界

当前展示应聚焦 v1 能力，不夸大为完整实验平台。第一版不强调登录、多用户协作、在线部署、历史项目管理或外部 issue 系统集成。

对尚未实现或计划中的能力，文档中应标注为 `后续计划` 或 `规划能力`，避免 GitHub 读者误解当前完成度。
