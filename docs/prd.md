# Local A/B Test Workbench PRD

## 一. 产品定位

Local A/B Test Workbench 是一个面向产品增长场景的本地 A/B 测试分析工作台。它把原本依赖脚本、Excel 和人工解释的实验分析流程，收敛为一个可复用的单页 Web 工具。

核心流程：

```text
上传 Excel -> 识别字段 -> 选择分组和指标 -> 执行显著性分析 -> 生成图表 -> 查看结论 -> 下载结果
```

## 二. 目标用户

- 产品经理：需要快速判断实验是否值得上线，并理解风险提示。
- 增长 / 运营同学：需要用 Excel 数据完成常见 A/B 实验复盘。
- 数据分析同学：需要把标准分析流程沉淀为可复用工具，减少重复脚本操作。
- 项目展示读者：需要快速理解项目如何把数据分析能力产品化。

## 三. 需求背景

当前 A/B 测试分析能力主要由脚本和本地文件流转完成，使用者需要理解输入文件格式、运行命令、输出 workbook 和图表目录。该流程对非技术同学不够友好，也容易遗漏数据质量检查。

本工具的目标是把核心流程产品化：

- 降低 Excel 实验数据分析门槛。
- 明确展示数据质量 Gate 和 WARNING。
- 输出 PM 可读结论，而不只输出统计数值。
- 将分析结果和图表沉淀为可下载、可分享的本地文件。

## 四. 成功指标

v1 可认为成功，当满足以下条件：

- 用户可以在本地完成一次从 Excel 上传到结果查看的完整分析。
- 用户无需手动运行统计脚本即可获得 p 值、置信区间、提升幅度和显著性判断。
- Gate 状态能阻止明显不可靠的结果继续出图。
- 图表生成只读取分析结果，不重新计算显著性。
- README 和文档能让 GitHub 读者在 5 分钟内理解项目价值、技术结构和启动方式。

## 五. 功能范围

### 5.1 Excel 上传和字段识别

用户上传 `.xlsx` 或 `.xlsm` 文件后，系统需要：

- 保存原始文件到本地 `storage/uploads/{upload_id}/original.xlsx`。
- 读取 workbook sheet 列表。
- 默认选择第一个 sheet。
- 自动识别表头行。
- 展示字段名、字段索引、字段类型和样例值。
- 提供预览行，帮助用户确认字段含义。

字段类型第一版识别为：

- `binary`：0/1 或布尔值。
- `number`：连续数值。
- `string`：文本或类别字段。
- `empty`：没有有效样例值。

### 5.2 分析配置

用户需要选择：

- 1 个分组字段，作为 A/B 分组字段。
- 1 到 5 个分析指标，作为因变量。
- 0 到 2 个可选分层字段，用于分层诊断或 ANOVA 相关分析。

约束规则：

- 分组字段不能同时作为指标字段。
- 分层字段不能同时作为指标字段。
- 第一版只支持两组对比。
- 指标字段最多 5 个。

### 5.3 显著性分析

分析执行后，系统输出：

- Gate 状态：`PASS` / `PASS_WITH_WARNING` / `FAIL`。
- 数据质量检查。
- 样本量统计。
- 主指标检验结果。
- 可选分层 / ANOVA 结果。
- 观察指标表。
- PM 可读结论。
- 可下载的 Stage 2 workbook。

不同指标类型的处理：

- 二元指标：两样本比例检验，输出比例差异、p 值和 95% 置信区间。
- 连续数值指标：均值差异检验，输出均值差异、p 值和 95% 置信区间。
- 类别指标：卡方检验，输出列联表、卡方统计量、自由度和 p 值。
- 不支持指标：标记为不可计算或不可出图，不阻断其他可支持指标。

### 5.4 质量 Gate

Gate 规则：

- `PASS`：硬性检查通过，未发现明显风险。
- `PASS_WITH_WARNING`：硬性检查通过，但存在样本不均衡、字段风险或分层风险。
- `FAIL`：无法可靠分析，例如分组字段不是两组对比。

交互规则：

- `FAIL` 结果不能生成图表。
- `PASS_WITH_WARNING` 可以生成图表，但需要显眼展示风险。
- WARNING 不能被包装成正向结论。

### 5.5 图表生成

图表生成基于 Stage 2 workbook，不重新计算统计结果。

输出规则：

- 每个可出图指标生成 1 张指标图。
- 生成 1 张汇总图。
- 生成 1 张完整图表合集。
- 指标最多 5 个，因此图表上限为 7 张。
- 不可出图指标写入 skipped metrics。

图表需要展示：

- 指标名。
- A/B 值。
- 差异。
- p 值。
- 显著性。
- 必要的统计补充信息。

### 5.6 调试样例数据

首页保留调试样例入口，便于没有真实 Excel 时快速验证流程。

当前样例重点支持：

- 卡方检验示例数据。
- 两组实验样本。
- 类别指标和二元 / 数值指标混合场景。

### 5.7 问题上报

规划能力：用户可以填写问题标题、描述和是否稳定复现，系统自动附带上下文并写入本地报告。

报告内容应包含：

- 输入文件名。
- sheet 名。
- 分组字段。
- 指标字段。
- Gate 状态。
- WARNING 列表。
- 输出文件路径。
- 时间戳。

当前根目录代码暂未接入 `/api/reports` 路由；旧打包目录中存在 report 相关实现，可作为后续合并参考。

## 六. 页面结构

页面采用单页仪表台，不做分步向导。

页面区块：

- 顶部状态栏：当前文件、分析状态、图表状态、Gate 状态。
- 文件与字段配置区：上传 Excel、生成 demo、sheet 切换、字段预览。
- 分析配置区：选择分组字段、指标字段、可选分层字段。
- 主操作区：执行分析、生成图表。
- 结果区：分析摘要、统计表、PM 结论、图表预览。
- 下载区：复制 workbook 链接或图表链接。

## 七. 非功能需求

- 本地优先：所有上传和结果默认写入本地 `storage/`。
- 可重复执行：每次上传和分析生成独立 id。
- 风险可见：Gate 和 WARNING 必须在页面上清晰呈现。
- 结果可追踪：分析请求、状态、workbook、结构化 JSON 和图表 manifest 均应落盘。
- 隐私友好：v1 不依赖外部服务，不上传业务数据到第三方系统。

## 八. 暂不支持

v1 不做：

- 登录和权限。
- 多用户协作。
- 外部 issue 系统接入。
- 历史项目管理。
- 页面内编辑图表。
- 在线数据库存储。
- 多组实验的完整事后检验。
- 自定义统计检验规则。
